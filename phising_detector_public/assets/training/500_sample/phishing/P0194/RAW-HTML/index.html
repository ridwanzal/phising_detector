  • Game over

    Anterior Siguiente

    Para terminar el juego sólo nos falta gestionar el final de partida. Terminaremos la partida cuando un asteroide choque contra la nave. En ese momento mostraremos un mensaje de game over y ofreceremos la opción de volver a empezar.

    Para visualizar el final de partida vamos a crear una capa donde incluiremos dos textos, uno con el texto “Game Over” y otro con “Press spacebar to start”. Pulsa el botón para ir al editor de escenas y crea una nueva capa llamada game over. En ella crea un objeto de texto y llámalo game over message. Modifica sus propiedades con los siguiente valores:

    • fontSize = 120
    • text = Game Over
    • fill color:
      • r = 255
      • g = 0
      • b = 0
    • anchor = CENTER
    • x = 515
    • y = 331
    • scaleX = 0,1
    • scaleY = 0,1

    Como habrás apreciado, hemos puesto una escala del 0,1 tanto en el eje X como en el Y. La escala acepta valores entre 0 y 1, donde 0 significa sin tamaño y 1 significa el 100% del tamaño original. El 0,1 representa el 10%, por tanto hemos escalado el texto al 10% de su tamaño original. La razón es que vamos a realizar una animación para que el texto escale desde el 10% hasta el 100% de su tamaño. Así parecerá que el texto viene desde el fondo.

    Vamos a crear un segundo texto que pondremos debajo del game over message indicando que se debe pulsar la tecla espacio para comenzar una nueva partida. Por lo tanto, crea un nuevo objeto de texto y llámalo restart. Como vamos a realizar la misma animación a los dos textos, es conveniente que restart sea hijo de game over message, porque animando al objeto padre, se animarán ambos. Entonces, en el panel Asset Manager arrastra restart sobre game over message. Verás que hereda las propiedades de escalado del padre, quedándose a una escala del 10%. Más tarde, cuando cambiemos la escala de game over message, también cambiará la de sus hijos, es decir, la de restart. Modifica las siguientes propiedades de restart:

    • fontSize = 40
    • text = Press space to start
    • anchor = CENTER
    • x = 0
    • y = 74
    • visible = false

    La última propiedad pone el objeto restart como invisible. Esto lo hacemos para que cuando realicemos la animación del texto viniendo desde el fondo de la pantalla, no se muestre el segundo texto hasta que lo indiquemos expresamente a través de la lógica que programaremos a continuación.

    Para terminar con el diseño de los elementos visuales, haz click en la capa game over en el panel de Asset Manager, y pon su propiedad visible a false. Así los elementos de la capa no se verán hasta que nosotros la pongamos visible cuando detectemos que la nave colisiona con un asteroide.

    Volvamos a la lógica pulsando . Lo primero que tenemos que hacer es detectar en qué momento la nave colisiona con un asteroide. Como esto ya lo tenemos resuelto en el script Spaceship ➝ Collisions ➝ Spaceship-Asteroids, vamos a simplemente notificarlo al exterior mediante un conector de salida. Arrastra entonces un conector de salida y llámalo collision. El punto en el que tenemos la certeza de que la colisión ha tenido lugar es el trigger true de BoundingCollision, por lo que conecta el conector que acabas de crear a este trigger.

    Sube un nivel y crea otro conector de salida con el nombre spaceshipDestroyed. Conéctalo al trigger collision que acabamos de crear en el paso anterior.

    Sube otro nivel más. En este nivel estamos en el script Spaceship. Para ordenar mejor la lógica de nuestro juego, vamos a notificar hacia el exterior la colisión de la nave para que sea otro script el que gestione qué hacer. Por lo que una vez más, crea otro conector de salida con el mismo nombre spaceshipDestroyed y conéctalo con el trigger del script Collisions del mismo nombre.

    Sube un nivel más, para llegar al nivel raíz. Aquí ya tenemos un script Spaceship con un nuevo trigger spaceshipDestroyed que nos indica cuando la nave es destruida. Para mantener limpia la lógica crea un nuevo script al que vamos a llamar Game over. Conecta el trigger spaceshipDestroyed con el activador in1 de Game Over.

    Entra en Game over. Primero renombra el conector de entrada in1 por show. Como hemos mencionado antes, cuando la nave se destruya, vamos a mostrar un cartel de “Game over” con una animación que escale desde el 10% de su tamaño (lo hemos dejado a ese tamaño en el editor de escenas), hasta el 100%, en un segundo de tiempo. Y cuando esté al 100% haremos visible el segundo mensaje para volver a empezar una nueva partida.

    Empecemos haciendo visible la capa game over que es la que contiene los dos textos que vamos a mostrar. Arrastra la capa desde el panel de AssetManager para crear un ActionOnParam y elige la acción setVisibility. Haz doble click sobre el parámetro visibility para crear una variable y dale el valor true en sus propiedades. Conecta la caja al conector de entrada.

    Ahora vamos a realizar la animación. Consiste en pasar el objeto texto game over message desde su escala del 10% hasta una escala del 100% en un un segundo de tiempo. Para ello usaremos la caja MoveFromTo que ya hemos utilizado anteriormente, pero en este caso vamos a configurarla para que modifique la escala en lugar de la posición. Arrastra un MoveFromTo desde la categoría Animation en la pestaña Blackboxes. Haz click sobre ella para mostrar sus propiedades, y modifica translate a false y scale a true.

    Ahora vamos a completar los parámetros:

    • target: arrastra el objeto game over message desde el panel de Asset Manager a este parámetro.
    • seconds: crea una variable y asígnale el valor 1.
    • fromScale: crea una variable y asigna a las propiedades x e y el valor 0,1, es decir, x=0,1 e y=0,1.
    • toScale: crea una variable y asigna a las propiedades x e y el valor 1, es decir, x=1 e y=1.

    Ahora tenemos que hacer visible el objeto restart cuando la animación termine (tardará un segundo). Arrastra el objeto restart desde el panel Asset Manager y selecciona su acción setVisibility. Crea una variable en el parámetro visibility y cambia su propiedad content a true. El trigger arrived de MoveFromTo nos informa de la finalización de la animación, por lo que lo puedes conectar al activador do.

    Ahora tenemos que escuchar a la tecla espacio para que cuando sea pulsada, reiniciar la partida. Usaremos la caja KeyboardListener que está en la categoría Device en la pestaña de Blackboxes. Si haces click sobre ella verás que tiene muchas propiedades, una por cada tecla del teclado. Pon la propiedad Space a true. Conecta el trigger done de la caja anterior al on de KeyboardListener.

    Cuando pulsen la tecla espacio se disparará el trigger Space. Lo primero que deberíamos hacer es desactivar KeyboardListener para que no siga escuchando a la tecla, porque recordad que estamos escuchando esta misma tecla para disparar con la nave. Si la dejáramos activa, cada vez que se pulsara el espacio tras el primer game over, la nave realizaría un disparo y volvería a comenzar el juego. Por lo tanto tenemos que conectar el trigger Space con el activador off en la misma caja KeyboardListener.

    Y por último vamos a notificar al exterior que han pulsado el espacio, para lo que usaremos un conector de salida. Arrastra uno y llámale restart. Finalmente conéctalo al trigger Space.

    NOTA: Puedes conectar múltiples entradas y salidas tanto a activadores como a triggers. Además puedes seleccionar el orden en el que se llamarán haciendo click en ellas y modificando su propiedad call order.

    Sube un nivel. En el nivel raíz verás que Game Over tiene un trigger reset que todavía no hemos conectado a nada.

    Si haces un preview en este momento, puedes comprobar que cuando chocas la nave contra un asteroide, sale el cartel de Game Over con una animación y a continuación se muestra el segundo cartel de pulsar el espacio para volver a comenzar.

    No obstante, si pulsas el espacio, la nave, aunque esté oculta, sigue disparando. Y si un asteroide choca contra ella, a pesar de ser invisible, se vuelve a realizar la animación de Game Over. ¿Qué está pasando? Pues como habrás intuido, como el script Spaceship sigue activo, sus scripts de colisiones, control de teclado y creación de disparos siguen funcionando, por lo que hay que detenerlo. Recordad que ya hemos previsto esta situación incluyendo un conector off que desconecta todos los scripts incluidos en Spaceship, por lo que vamos a utilizarlo. Conecta el trigger spaceshipDestroyed al activador off del mismo script.

    Haz un preview y comprueba que tras chocar contra un asteroide y salir el Game Over, si pulsas el espacio, la nave no dispara, ni tampoco colisiona con los asteroides.

    Llegados a este punto, tan sólo nos falta iniciar una nueva partida cuando se lance el trigger restart del script Game Over. Quizá te veas tentado de conectar restart al activador on de Spaceship, no obstante tendríamos que realizar algunas modificaciones para que funcionara correctamente. Mejor que eso, vamos a crear un nuevo script en donde incluiremos todas las inicializaciones previas a comenzar una nueva partida. Esto es muy común en la creación de juegos y en la programación en general, y nos ayudará a mantener el código ordenado.

    Crea un script y llámalo init. Entra en él y cambia el nombre del conector in1 a do. Cuando comencemos una partida queremos que la escena y sus elementos se inicien con los valores que les hemos dado cuando los hemos diseñado. Para hacer esto arrastra la escena game desde el panel de Asset Manager (la escena game está bajo el elemento scenes, no en layers que es la primera que se ve). Como siempre se crea un ActionOnParam donde debes seleccionar la acción reset.

    A continuación vamos a vaciar las dos colecciones shots y asteroids, porque después de la primera partida contendrán elementos obsoletos que no nos interesan. Vamos a crear primero un script para inicializar la colección shots. Crea un script y llámalo Reset shots. Entra en él y renombra el conector in1 a reset. Añade un conector de salida y llámalo done. Vuelve a este nivel y conecta el trigger done de ActionOnParam con el activador reset.

    Vuelve a entrar en Reset shots. Para recorrer la colección volveremos a usar la caja ForEach, que está bajo la categoría Collections en la pestaña Blackboxes. Se trata de eliminar todos los elementos contenidos en la colección y finalmente vaciar al colección, para garantizar que cada partida comienza con una colección de disparos vacía.

    Arrastra una caja ForEach al panel y conecta el conector reset con el activador start.

    Ups! Necesitamos la colección shots pero no la tenemos en este script, por lo que tendremos que compartirla para poder tener acceso a ella. Vete al script Spaceship donde tenemos acceso a ambas colecciones, y arrastra la colección shots a la barra de compartir.

    Ahora sube un nivel y comparte ambas colecciones con el script Init. Recuerda que se comparte arrastrando las variables al script donde las quieres usar. Por tanto, arrastra shots y asteroids desde el script Spaceship hasta Init.

    Ahora que ya tenemos las dos colecciones que necesitamos en el script Init, entra para seguir con la lógica.

    Deberías ver las dos colecciones en la barra de compartir. Arrastra la colección shots al script Reset shots.

    Entra en Reset shots y arrastra la colección shots desde la barra de compartir al parámetro de entrada target de la caja ForEach. Recuerda que esta caja recorre todos los elementos de la colección depositando en cada paso un elemento en el parámetro de salida item y lanzando el trigger step. Haz doble click en el parámetro item para crear una variable y recoger cada disparo. Luego arrástralo al panel para crear un ActionOnParam y selecciona la acción delete. Conecta el trigger step con el activador do.

    Una vez que hemos eliminado los disparos, tenemos que vaciar la colección porque contiene elementos que han sido destruidos. Arrastra la variable shots de la barra de compartir (o del parámetro target) y suéltala en el panel para crear un ActionOnCollection. Selecciona la acción empty. Finalmente conecta el trigger finished de ForEach con el activador do de ActionOnCollection, y el trigger done de este último al conector de salida done.

    Sube un nivel. Ahora tenemos que hacer exactamente lo mismo con la colección asteroids. Podemos repetir los mismos pasos, poniendo utilizando en esta ocasión la colección asteroids. No obstante, vamos a hacer una copia del script Reset shots. Haz click con el botón derecho sobre Reset shots y selecciona la opción copy.

    Luego haz click de nuevo con el botón derecho sobre el fondo y selecciona paste. Verás que aparece un nuevo script con el mismo nombre y mismos parámetros. Modifica el nombre del nuevo script por Reset asteroids y conéctalo al trigger done de Reset shots.

    En Reset asteroids necesitamos la colección asteroids, pero como hemos copiado el de los disparos, tenemos shots. Haz click con el botón derecho sobre la variable shots de Reset asteroids y selecciona la opción Remove.

    Ahora arrastra la colección asteroids que está en la barra de compartir y suéltalo sobre el script Reset asteroids.

    Comprueba que ahora el script tiene la colección asteroids. Entra en el script Reset asteroids. En este script lo que queremos hacer es vaciar la colección asteroids. Para ello vamos a sustituir las colecciones shots en los parámetros target de las cajas ForEach y ActionOnCollection, por asteroids. Arrastra asteroids desde la barra de compartir sobre el parámetro target de ForEach. Verás que asteroids sustituye a shots. Es decir, si soltamos una variable sobre un parámetro que ya contiene una variable, ésta última será sustituida por la que estamos soltando. Vuelve a arrastrar la variable asteroids sobre el parámetro target de ActionOnCollection.

    Sube un nivel. Vamos a terminar el script de init añadiendo un conector de salida. Arrastra un conector de salida desde la pestaña Connectors y llámale done. Conecta el trigger done de Reset asteroids al conector de salida.

    Y por fin estamos en el último paso para terminar nuestro juego. Sube un nivel y vamos a realizar los últimos cambios antes de compartir con el mundo nuestro juego.

    Al comienzo de este tutorial hemos unido el evento inicial gameStarted con los scripts Background, Spaceship y Asteroids. Ahora lo que queremos hacer es que lo primero que se ejecute sea el script Init, y cuando éste termine, llame a los demás. Elimina las conexiones que salen de gameStarted. Para eliminar una conexión, haz click con el botón derecho sobre una conexión y selecciona Delete. Conecta gameStarted al activador do de Init. Y a continuación conecta el trigger done de Init con los activadores on de Background, Spaceship y Asteroids.

    Ahora lo único que nos falta para terminar es reiniciar todo cuando el usuario pulse la barra de espacio, es decir, cuando se lance el trigger restart del script Game Over. Conecta el trigger restart con el activador do de Init.

    También puedes consultar el videotutorial que explica esta fase de la creación del juego.
